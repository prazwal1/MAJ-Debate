%% ── Abstract ────────────────────────────────────────────────────────────────
\begin{abstract}
We propose \textbf{MAJ-Debate}, a multi-agent LLM system that simulates
full debates through four modular stages: persona-driven side-picking agents
generate stance-specific arguments, an attack-relation brain labels every
argument pair as \texttt{Attack}/\texttt{Support}/\texttt{Neutral} with
natural-language justifications, a Dung argumentation framework engine
computes grounded and preferred extensions over the resulting directed graph,
and a judge brain produces an explainable verdict. Unlike prior work that
addresses only counter-argument generation or pairwise relation detection
in isolation, our pipeline closes the \textit{end-to-end judgment gap}.
We use DebateSum (\DebateSumArgs{} arguments) as a few-shot retrieval corpus
for argument generation and ``Which Side Are You On?'' (\WhichSideArgs{}
annotated examples) for argument strength calibration; neither dataset
provides debate-level winner labels. Judgment accuracy is evaluated against
the DDO dataset~\citep{durmus2019ddo} (\DDOTotal{} crowd-voted debates,
\DDOSample-debate evaluation sample) as an external benchmark, supplemented
by a new \HumanTestTopics-topic human-judged test set. We treat DDO labels
as a measure of \textit{audience-level persuasion} on a public platform,
and use the human test set to approximate \textit{adjudicated logical
correctness} where such labels are available. We further analyse cases
where persuasion and correctness disagree, to interpret whether
MAJ-Debate's graph-grounded verdicts align more closely with rhetorical
convincingness or logical validity. Ablation studies isolate the
contribution of each component; we hypothesize that the full graph-grounded
pipeline will substantially outperform single-LLM baselines in both
persuasion agreement and explainable correctness-oriented judgment.
An interactive demo and open-source code are released to advance
reproducible agentic argumentation research.
\end{abstract}

%% ── 1. Introduction ─────────────────────────────────────────────────────────
\section{Introduction}

Automated debating has advanced rapidly with large language models~(LLMs),
yet closing the full debate cycle, from argument generation to structured
judgment, remains an open challenge. Human debating involves dynamic,
multi-perspective clashes of ideas over multiple turns, requiring a system
to construct attacks, weigh competing claims, and resolve cycles of
counter-argument; these are capabilities that current systems do not provide.

Existing LLM approaches stop at one of three sub-tasks: essay generation,
pairwise counter-argument production, or stance detection. None constructs
an explicit attack graph and delivers a formal, \textit{explainable} judgment
of which side wins. This gap leaves automated systems unable to handle
conflicting evidence, implicit premises, or long-context continuity.

We therefore propose \textbf{MAJ-Debate}, a persona-driven multi-agent
framework. Four to six LLM agents independently pick opposing sides and
generate arguments; a dedicated attack brain labels every argument pair
with natural-language justifications; a graph engine computes grounded and
preferred extensions following Dung's abstract argumentation
semantics~\citep{dung1995}; and a judge brain outputs the winning side,
a confidence score, and the decisive ``killing'' attacks. The system
addresses three research questions:

\begin{itemize}
  \item \textbf{RQ1:} Does an explicit Dung argumentation graph improve
        automated judgment accuracy relative to prompt-only LLM baselines?
  \item \textbf{RQ2:} Does increasing the number of debating agents (2
        vs.\ 5+) produce more diverse and persuasive arguments, as measured
        by attack diversity and human persuasiveness ratings?
  \item \textbf{RQ3:} How does targeted implicit-premise attack modeling
        compare to zero-shot pairwise prompting in identifying key argument
        clashes?
\end{itemize}

eight-configuration ablation, enabling applications in education, policy
analysis, and agentic AI research.

%% ── 2. Related Work ─────────────────────────────────────────────────────────
\section{Related Work}

\paragraph{Datasets}
DebateSum~\citep{roush2020debatesum} provides \DebateSumArgs{}
evidence-argument-summary triples from competitive policy debate, used in
MAJ-Debate as a few-shot retrieval corpus for Stage~1 argument generation;
it contains no debate-level winner labels.
``Which Side Are You On?''~\citep{li2024whichside} provides \WhichSideArgs{}
argument-level convincing-ness rankings used to calibrate Stage~2 strength
scores; it likewise contains no debate-level judgment.
ARIES~\citep{gemechu2024aries} offers a standardized benchmark for argument
relation identification, revealing severe cross-domain generalization gaps.
\citet{ivanova2024lets} characterize argument quality dimensions directly
relevant to our persuasiveness metric.
The DDO dataset~\citep{durmus2019ddo} comprises \DDOTotal{} debates from
Debate.org across \DDOTopics{} topics, each with a crowd-voted winner
label under the criterion of ``making more convincing arguments,'' making
it the only publicly available large-scale corpus with debate-level human
judgment. We use a \DDOSample-debate sample as our primary external
evaluation benchmark.

\paragraph{Argument generation}
Debate-to-Write~\citep{hu2025debate2write} introduces persona-driven
multi-agent debate to improve argument diversity, the closest prior work
to our Stage~1. LLM Debate Opponent~\citep{ozaki2025llmdebate} focuses on
implicit-premise attacks, directly informing our Stage~2 design.
Debate-Reflect-Distill~\citep{zhou2025debatereflect} organises multi-turn
debates for preference-optimization distillation.
\citet{yeginbergen2025dynamic} augment counter-argument generation with
dynamic knowledge retrieval, and \citet{bao2025quality} explore
quality and diversity trade-offs in synthetic argument data.

\paragraph{LLMs and argumentation}
\citet{chen2024exploring} survey LLM potential across computational
argumentation tasks. \citet{bezou2025largellm} probe whether LLMs understand
argument schemes, finding substantial limitations.
\citet{wangchen2024interpretability} study interpretability in collaborative
classroom argumentation.

\paragraph{Gaps addressed}
Four gaps remain: (1)~no prior system constructs an explicit attack graph
and computes formal argumentation extensions for judgment; (2)~multi-agent
systems improve generation but lack principled winning-side determination;
(3)~relation-labeling components degrade in end-to-end settings due to
error propagation; and (4)~prior work lacks an external, crowd-validated
judgment benchmark, relying instead on in-house annotation. MAJ-Debate
addresses all four by pairing the graph-grounded pipeline with DDO as an
independent evaluation signal.

%% ── 3. Architecture ─────────────────────────────────────────────────────────
\section{Architecture}

MAJ-Debate consists of four LLM-powered stages illustrated in
Figure~\ref{fig:pipeline}.

\begin{enumerate}
  \item \textbf{Stage 1 - Side-Picking Agents}~\citep{hu2025debate2write}:
        two to three Pro agents and two to three Con agents independently
        generate three to five stance-specific arguments per topic.
        Each agent is instantiated with a \textit{structured persona template}
        that specifies three controlled dimensions plus an optional
        \textit{demographic-conditioning} field: (i)~a \textit{reasoning
        style} (e.g., logical-empirical, ethical-normative, or
        economic-consequentialist), (ii)~a \textit{rhetorical constraint}
        (e.g., cite evidence, appeal to principle, or challenge implicit
        premises), and (iii)~a \textit{stance label} (Pro or Con).
        When enabled, demographic conditioning is implemented as a brief
        neutral framing stub (e.g., ``first-generation student advocate'')
        that can influence argument emphasis without changing the topic.
        Example personas include \textit{Rationalist Pro} (logical-empirical,
        evidence-first) and \textit{Rights Defender Con}
        (ethical-normative, principle-based).
        Persona categories are held fixed across all experimental configurations
        so that agent diversity constitutes a controlled independent variable
        rather than an incidental design choice, directly enabling the
        comparison in RQ2.

        To ground agent output in real competitive debate language,
        the top-$K$ BM25-retrieved arguments from DebateSum on the target
        topic are injected into each persona prompt as style demonstrations
        before generation begins. This leverages DebateSum as a
        \textit{few-shot retrieval corpus} without requiring fine-tuning.

        Stage~1 proceeds in two sub-rounds to ensure that attack relations
        in Stage~2 are grounded in genuine opposition rather than
        post-hoc inference.
        In \textit{Sub-round 1}, all agents generate their initial arguments
        independently, without access to the opposing side's output.
        In \textit{Sub-round 2}, each agent receives the complete set of
        opposing arguments as read-only context and produces up to two
        targeted counter-arguments, each explicitly referencing the specific
        opposing claim it addresses.
        This two-round design guarantees that the argument pool contains
        both independent claims and intentional attacks, providing
        Stage~2 with structurally grounded material rather than requiring
        it to retrofit attack relations onto arguments that were never
        constructed as responses to one another.

  \item \textbf{Stage 2 - Attack-Relation Brain}~\citep{ozaki2025llmdebate}:
        every ordered argument pair $(A_i, A_j)$ is classified as
        \texttt{Attack}, \texttt{Support}, or \texttt{Neutral} with a
        one-sentence justification grounded in the specific claims made by
        each argument. Each argument is also assigned a continuous
        \textit{strength score} $s_i \in [0,1]$ derived from the LLM's
        confidence and validated against the convincing-ness rankings from
        ``Which Side Are You On?''~\citep{li2024whichside}, which serves as
        the \textit{strength calibration reference} for this stage.
        A confidence threshold of $0.65$ filters low-certainty labels to
        mitigate hallucination, and a \texttt{None} option prevents forced
        labeling of unrelated pairs.
        For $N$ total arguments across both sub-rounds, this stage requires
        $N(N-1)$ ordered comparisons; at four agents with three initial
        arguments and two counter-arguments each ($N \approx 20$),
        the \DDOSample-debate evaluation set incurs approximately
        76{,}000 API calls in total, mitigated in practice by the
        top-$K$ BM25 retrieval filter described in Section~\ref{sec:data}.

  \item \textbf{Stage 3 - Argumentation Framework Engine}: labeled
        relations are encoded as a directed graph with adjacency-list
        structure. Grounded and preferred extensions are computed following
        Dung's semantics~\citep{dung1995}. Cyclic attack structures and
        disconnected subgraphs are handled explicitly; when no grounded
        extension selects a clear winner, the system falls back to a
        majority vote of preferred extensions, and ties are flagged as
        \textit{undecided}.

  \item \textbf{Stage 4 - Judge Brain} (LLM-as-Judge): the winning
        extension identified by Stage~3 is passed to a final LLM judge,
        which serves a strictly \textit{interpretive} rather than
        \textit{decisional} role. Critically, the judge does not re-evaluate
        arguments from scratch; it receives the grounded extension as a
        hard constraint and is tasked only with producing a natural-language
        explanation of why the extension-selected side prevails, identifying
        the decisive killing attacks, and assigning a Likert persuasiveness
        rating. This design ensures the formal graph determines the verdict
        while the LLM contributes the explainability layer that a pure graph
        algorithm cannot provide. The judge is prompted to cite specific
        argument texts from the extension, so any deviation from the
        graph-determined outcome is detectable and logged as a Stage~4
        hallucination event in the error analysis.
\end{enumerate}

The pipeline is orchestrated with \texttt{LangChain} and tracked with
Weights~\&~Biases and MLflow. A web demo enables interactive exploration
of generated debates, the argument graph, and final judgments.

%% ── 4. Dataset and Exploratory Analysis ────────────────────────────────────
\section{Dataset and Exploratory Analysis}
\label{sec:data}

We use three public debate corpora in strictly separated roles and construct
a new human-judged test set (Table~\ref{tab:datasets}). DebateSum and
``Which Side Are You On?'' are \textit{pipeline input datasets} with no
winner labels; DDO is the \textit{external judgment benchmark} providing
crowd-sourced ground truth independent of our annotation. All processing
steps and EDA figures are reproducible via
\texttt{MAJ\_Debate\_EDA\_Pipeline.ipynb}.

\paragraph{DebateSum~\citep{roush2020debatesum}}
\DebateSumArgs{} evidence-argument-summary triples spanning \DebateSumTopics{}
unique topics with a mean of \AptMean{} arguments per topic
(median~\AptMedian, $\sigma{=}\AptStd$). The distribution is right-skewed
(Figure~\ref{fig:debatesum}): a small number of heavily-debated policy
resolutions dominate the corpus.
\textit{Role.} Few-shot retrieval corpus for Stage~1. The top-$K$
BM25-matched arguments for each debate topic are injected into persona
prompts as style demonstrations. No winner labels are present.

\textit{Preprocessing.} Arguments are truncated at sentence boundaries
within \TruncLimit{} tokens (affecting \PctArgsTrunc\% of all arguments;
mean raw length \MeanRawTokens{} tokens, post-truncation mean
\MeanTruncTokens{} tokens). HTML artifacts are stripped and topics with
fewer than \MinArgPairs{} argument pairs are removed to guarantee graph
connectivity in Stage~3, retaining \NTopicsKept{} topics
(\PctTopicsKept\% of the corpus).

\paragraph{Which Side Are You On?~\citep{li2024whichside}}
\WhichSideArgs{} annotated examples across \WhichSideTopics{} topics.
Normalised quality scores have mean \QualityMean{} ($\sigma{=}\QualityStd$)
after mapping to $[0,1]$ (Figure~\ref{fig:whichside}). Stance labels are
approximately balanced (ratio~\StanceBalance).
\textit{Role.} Strength calibration reference for Stage~2. The
human-annotated convincing-ness rankings provide an external signal against
which LLM-assigned argument strength scores $s_i$ are validated. No
debate-level winner label is present.

\paragraph{DDO Judgment Benchmark~\citep{durmus2019ddo}}
The Debate.org (DDO) corpus comprises \DDOTotal{} debates across
\DDOTopics{} topics collected over ten years with crowd-voted winner
labels under the criterion of ``making more convincing arguments.''
Of these, \DDOComplete{} debates are structurally complete (non-tie,
$\geq\DDOModalRounds{}$ rounds). Across the full corpus, \DDOProWin\% of
debates are won by the Pro side, \DDOConWin\% by Con, and \DDOTie\% end
in a tie (Figure~\ref{fig:ddo_winners}).
\textit{Role.} External judgment benchmark. We sample \DDOSample{} debates
balanced across topics (Figure~\ref{fig:ddo_sample}); within this sample,
\DDOSamplePro\% are Pro wins and \DDOSampleCon\% are Con wins.
MAJ-Debate is run on each topic and its verdict is compared against the
crowd vote, providing an independent signal of \textit{public
persuasiveness}. Importantly, we do not interpret DDO agreement as pure
logical correctness; where adjudicated logical labels are available, we
report correctness agreement separately and analyse disagreement cases.

\paragraph{New Human Test Set}
\HumanTestTopics{} debate topics spanning policy, ethics, and science, each
annotated by three independent judges drawn from the project team's academic
network (Figure~\ref{fig:domains}). This constitutes a convenience sample;
results should be interpreted as within-group estimates and cross-group
validation is identified as future work. The DDO benchmark mitigates this
limitation by providing an independent external signal. Domain proportions
are balanced to reduce the policy-heavy skew in DebateSum
(${\approx}42\%$ policy) toward a more representative distribution
(${\approx}35\%$ policy, $25\%$ ethics, $15\%$ science, $15\%$ society).
Inter-annotator agreement is reported as Krippendorff's~$\alpha$.

%% ── 5. Experimental Design ──────────────────────────────────────────────────
\section{Experimental Design}

\paragraph{Baselines}
We compare against three baselines: (i)~a \textit{single-LLM baseline} in
which one model generates arguments and returns a verdict without multi-agent
structure or a graph; (ii)~a \textit{chain-of-thought (CoT) baseline} that
prompts a single model to reason step-by-step before judging; and
(iii)~a \textit{GPT-4 direct judge} applied to the same argument pairs
without the Dung graph. These baselines enable clean isolation of the graph
contribution (RQ1) and the multi-agent contribution (RQ2).

In addition to the internal ablations and single-model baselines, we
perform direct head-to-head comparisons against two published end-to-end
systems that are most similar to our pipeline: (1)~\textbf{Debate-to-Write}
(Hu et al., 2025) -- persona-driven multi-agent generation followed by a
GPT-4 judge (no graph), and (2)~\textbf{LLM Debate Opponent} (Ozaki et al.,
2025) -- the full implicit-premise attack pipeline. Both systems are
re-implemented and evaluated on the exact same \DDOSample-debate DDO sample
and \HumanTestTopics-topic human test set, allowing direct comparison of
persuasion agreement (DDO) and correctness agreement (human) metrics. This
addresses the need for external validation against existing techniques and
provides a clearer picture of MAJ-Debate's relative performance.

\paragraph{Ablation configurations}
Starting from the single-LLM baseline, we incrementally add components
in eight configurations (Table~\ref{tab:ablation}), studying three factors:
(i)~agent count (2 vs.\ 5+), (ii)~attack modeling (zero-shot
vs.\ targeted premise attack~\citep{ozaki2025llmdebate}), and
(iii)~graph usage (none vs.\ full Dung extensions).

\paragraph{Metrics}
\begin{itemize}
  \item \textbf{Persuasion agreement (DDO)}: \% agreement with crowd-voted
    winner on the \DDOSample-debate external benchmark (audience
    persuasion / convincingness).
  \item \textbf{Correctness agreement (human)}: \% agreement with majority
    annotator label on the \HumanTestTopics-topic in-group test set,
    treated as an approximation of adjudicated logical correctness.
  \item \textbf{Persuasiveness}: Likert 1 to 5 human rating.
  \item \textbf{Attack diversity}: number of unique premises attacked.
  \item \textbf{Graph stability}: size of the grounded extension.
  \item \textbf{Disagreement \& regret}: conditional performance on
    subsets where crowd persuasion labels and adjudicated correctness
    labels agree vs. disagree; we report simple 0/1 regret (error rate)
    with respect to each label space and the regret gap between them.
\end{itemize}

Human evaluation on the in-group test set is conducted on 50 topics by
three annotators using a structured rubric. Inter-annotator agreement is
reported as Krippendorff's~$\alpha$; statistical significance with
${}^{*}p{<}0.05$ and ${}^{**}p{<}0.01$ (paired permutation test).
Metrics are logged with Weights~\&~Biases and replicated with MLflow.

%% ── 6. Expected Results & Analysis ─────────────────────────────────────────
\section{Expected Results and Analysis}

Table~\ref{tab:ablation} lists the eight planned configurations.
We hypothesize that the full MAJ-Debate pipeline will achieve
\FullAccLo{} to \FullAccHi\% agreement on both the DDO benchmark and the

%% ── 7. Discussion ───────────────────────────────────────────────────────────
\section{Discussion}
\label{sec:discussion}

\paragraph{RQ1 - Graph contribution}
The ablation comparison between the ``+~Dung Graph'' configuration and all
non-graph configurations isolates the graph's contribution. Accuracy is
measured primarily against DDO crowd votes, providing an external check
that is independent of our annotators. We expect grounded extensions to
resolve attack cycles that confound prompt-only judges, particularly in
closely-matched topics.

\paragraph{RQ2 - Agent diversity and additional agent factors}
Comparing 2-agent and 5-agent configurations in terms of attack diversity
and persuasiveness tests whether persona breadth surfaces non-obvious
premises. Because persona categories are held fixed across configurations
(Section~3), any observed difference is attributable to the number of
distinct reasoning styles instantiated, not to random prompt variation.
\citet{hu2025debate2write} demonstrate diversity gains from persona-driven
generation; we extend this to a judgment setting with a controlled design.
Beyond agent count and reasoning style, agent behavior may also be shaped
by \emph{demographic framing}, whereby agents are instantiated with
attributes such as age group, regional background, or occupational role
that could shift which implicit premises are surfaced and which values are
treated as shared. A related dimension is \emph{epistemic stance}: an
agent may default to empirical evidence, appeals to principle, or
challenges to the opponent's premises, and while this is partially
captured by our existing persona categories, it is not independently
varied. We identify both as open variables; the current
ablation isolates agent count as the primary factor, and demographic and
epistemic dimensions are deferred to future work. This scoping is reported
transparently so that RQ2 results are not over-generalized beyond
agent-count effects.

\paragraph{RQ3 - Implicit premise modeling}
Zero-shot prompting is compared against the targeted implicit-premise attack
approach of \citet{ozaki2025llmdebate}. We expect targeted prompting to
label a higher proportion of non-obvious cross-stance attacks.

\paragraph{Persuasion versus logical correctness}
MAJ-Debate is designed to optimize for \emph{structured argumentative
validity}, defined as whether the winning side holds more undefeated
arguments under Dung semantics, rather than audience persuasion. This distinction
matters because DDO crowd votes reflect which side a public audience found
more persuasive, which can diverge from what is logically stronger. We
therefore treat DDO agreement as a \emph{proxy signal} for judgment
quality rather than as ground truth for correctness. Systematic
disagreement between MAJ-Debate and DDO will be analysed by topic
category: cases where the crowd voted for the rhetorically vivid side
despite weaker argumentative structure are expected to constitute a primary
failure mode. The human test set, annotated with a rubric that explicitly
distinguishes logical strength from persuasiveness, provides a second
signal that decouples the two. Future work should investigate whether a
system optimising for Dung-grounded correctness can be adapted to predict
perceived persuasion when a persuasion-oriented output is required.

\paragraph{Computational scope}
The two-round Stage~1 design produces approximately $N \approx 20$ arguments
per topic. The top-$K$ BM25 retrieval step bounds Stage~2 comparisons to
at most $K^2$ pairs per topic with $K \leq 6$, reducing ordered pairs from
the unfiltered $380$ to $36$ per topic, giving $36 \times \DDOSample = 18{,}000$
API calls for the DDO evaluation set. The strength scoring adds at most
$20 \times \DDOSample = 10{,}000$ additional calls, keeping total Stage~2
cost well within budget for repeated ablation runs.

  \centering
  \includegraphics[width=0.9\columnwidth]{fig_complexity.png}
  \caption{Stage~2 API call growth with agent count (O($N^2$) ordered
  pairs). With BM25 filtering ($K{=}6$), pairs are capped at 36 per

  \label{fig:complexity}
\end{figure}

\paragraph{Limitations}
The human test set is annotated within a single academic network;
results from this set should be interpreted as within-group estimates.
The DDO benchmark mitigates this by providing an independent crowd-sourced
signal, but DDO winner labels reflect audience persuasion on a public
platform and may not correspond to logical correctness. Both limitations
are reported transparently and cross-group validation is identified as
future work. Dung semantics assume binary attack relations; graded
relations and cross-lingual debate are identified as high-value extensions.
Additionally, the current persona framework does not vary demographic
framing or fine-grained epistemic stance as independent factors; argument
diversity results should therefore be interpreted as lower bounds on what
a more fully varied agent pool might achieve.

\section{Summary of Progress}

\paragraph{Completed}
We have finalized the four-stage pipeline architecture including the
two-round argument generation design and the DebateSum few-shot retrieval
mechanism, selected and preprocessed all four datasets, configured the
Groq development environment with MLflow tracking, and implemented working prototypes of Stages~1 and~2 on ten debate
topics. The DDO \DDOSample-debate evaluation sample has been constructed
and verified for topic balance. All EDA figures including DDO analysis
in Section~\ref{sec:data} are fully reproducible via the accompanying
Jupyter notebook.

\paragraph{Remaining work}
Five milestones remain: (i)~integration of the Dung graph engine (Stage~3)
with cycle and disconnected-component handling; (ii)~implementation of the
judge brain (Stage~4) with grounding prompts; (iii)~execution of the full
ablation study with DDO as primary accuracy signal; (iv)~human evaluation
on 50 topics with structured rubric and agreement reporting; and
(v)~finalization of the interactive web demo.

\paragraph{Risks and mitigation}
\textit{Annotation bottleneck}: reduce to 30 topics if needed while
preserving domain balance.
\textit{API rate limits}: batch off-peak; fall back to Llama-3.1-8B.
\textit{Attack hallucination}: confidence filter and \texttt{None} option
implemented; model required to quote the attacked premise.
\textit{Graph complexity}: empty grounded extension falls back to
preferred-extension majority vote, documented as a distinct error category.
\textit{DDO label noise}: crowd votes reflect perceived persuasion, not
logical correctness; systematic disagreement between MAJ-Debate and DDO
will be analysed by topic category to distinguish genuine system errors
from label noise.

%% ── 9. Conclusion ───────────────────────────────────────────────────────────
\section{Conclusion}

MAJ-Debate introduces the first multi-agent, graph-grounded pipeline for
end-to-end automated debate judgment with explainable outputs. By
simultaneously addressing argument generation via DebateSum few-shot
retrieval, strength calibration via human convincing-ness annotations,
formal argumentation semantics, and LLM-based verdict production evaluated
against an independent crowd-sourced benchmark, the system bridges the gap
between isolated argument mining research and practical, deployable debating
systems. We release all code, prompts, and the new human-judged test set
to support reproducible research in agentic argumentation.
